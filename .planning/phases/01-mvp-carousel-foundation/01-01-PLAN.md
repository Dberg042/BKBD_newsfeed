## Objective

Build the core carousel display structure, data loading pipeline, and polling mechanism. This plan creates the HTML/CSS scaffold and JavaScript data layer that supports 15-second item rotation with automatic polling for content updates.

## Tasks

### Task 1: Create index.html with Tailwind layout and carousel structure

**Files:** index.html

**Action:** Create production HTML with DOCTYPE, meta viewport, Tailwind CSS v4 CDN from https://cdn.tailwindcss.com/v4, Google Fonts Inter, dark theme wrapper, header 80px with date/time elements, carousel container with image/headline/summary/source/QR placeholders, footer 120px with progress bar and queue preview, fallback error div, script tags for initialization.

**Acceptance criteria:** File exists, Tailwind v4 CDN configured with dark theme, all required IDs present (carousel-item, carousel-image, carousel-headline, carousel-summary, carousel-source, qr-code, progress-bar, queue-preview, fallback-state), opens without errors.

**Done:** index.html renders dark carousel scaffold.

### Task 2: Create test.html preview mode with PREVIEW MODE banner and Godkjenn button

**Files:** test.html

**Action:** Copy index.html to test.html. Add red PREVIEW MODE banner at top with dynamic date display. Add Godkjenn button top-right linking to GitHub editor for active.json.

**Acceptance criteria:** File exists, red banner with PREVIEW MODE text, Godkjenn button links to GitHub editor, structurally identical to index.html except for banner and button.

**Done:** test.html is preview mode with approval UI.

### Task 3: Create css/style.css with dark theme, TV-readable typography, animations

**Files:** css/style.css

**Action:** Create CSS with root color variables, global no-cursor/no-select styles, TV-readable typography (Headlines 64-72px, Summary 32-36px, Queue 24px), keyframe animations for progress bar (15s linear fill) and carousel fade (600ms total), header/footer layout, image fallback styling, warning indicator for errors.

**Acceptance criteria:** File exists, all color variables defined, typography sizes correct, animations keyframes defined, file loads without CSS parse errors.

**Done:** css/style.css provides dark theme and animations.

### Task 4: Create js/app.js with data loading, polling, error recovery

**Files:** js/app.js

**Action:** Create state management with lastKnownGood cache, fetchActiveDate() function with cache-busting, loadNewsContent() with parallel fetch and priority sorting, retry logic with exponential backoff (1s, 3s, 9s), startPolling() for 5-minute active.json checks, page reload on date change. When loadNewsContent() completes successfully and carousel exists, call window.carousel.init(newsData) to start rotation. If active.json date unchanged but news items differ from last load, call carousel.init(newItems) to refresh carousel with updated content. Error indicator after 3 consecutive failures, fallback to last known good state, initial load sequence.

**Acceptance criteria:** File exists, exports window.APP with required functions, fetches with cache-buster query params, implements exponential backoff retry, polls every 5 minutes, reloads on date change, refreshes carousel when news items change on same date, shows error indicator after 3 failures, never displays blank screen.

**Done:** js/app.js implements data pipeline with polling, error recovery, and carousel initialization.
