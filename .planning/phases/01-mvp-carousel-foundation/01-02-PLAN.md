## Objective

Create the carousel rotation engine, progress animation, and queue preview. This plan implements the 15-second item rotation loop, 600ms fade transitions, sample news data for testing, and integrates carousel initialization into app.js.

## Tasks

### Task 1: Create js/carousel.js with rotation timer and progress animation, and update js/app.js integration

**Files:** js/carousel.js, js/app.js

**Action:** Create carousel module with state (currentIndex, newsItems, timer, isAnimating). Exports: carousel.init(newsItems), carousel.next(), carousel.prev(), carousel.pause(), carousel.resume(). init() initializes carousel, renders first item, starts 15-second timer. next() moves to next item (wrap to 0 at end), triggers fade animation, updates progress bar, renders queue preview. Timer: setInterval every 15 seconds calling next(). Progress bar: CSS animation class applied on item load, 15s linear fill, resets when animation ends. Fade: Add/remove animation classes on carousel-item (carousel-fade-out/in) with 300ms each direction (600ms total). Queue preview: Extract next 3 items, truncate headlines to 30 chars max, display in id="queue-preview" with " → " separator. Handle edge cases: 5 items instead of 7, 0 items (fallback message), birthday card in rotation. Update js/app.js: After loadNewsContent() completes successfully, call window.carousel.init(newsData) to initialize carousel with loaded news items. Also handle birthday.json fetch with try/catch: if birthday.json returns 404, continue without adding birthday card (do not break carousel initialization).

**Acceptance criteria:** File exists, carousel.init() starts rotation, next() moves to next item, progress bar animates 15s, fade transition 600ms, queue preview shows next 3 items truncated to 30 chars, birthday card included in rotation if present, carousel.init() called from app.js after loadNewsContent(), 404 on birthday.json handled gracefully, handles fewer than 7 items, never gets stuck or frozen.

**Done:** js/carousel.js implements 15-second rotation with smooth transitions and queue display; app.js properly initializes carousel.

### Task 2: Create data/active.json pointer file

**Files:** data/active.json

**Action:** Create JSON file with structure { "date": "2026-05-13", "approved_at": "2026-05-13T06:00:00Z", "approved_by": "System" }. This is the single source of truth for which date folder the carousel displays.

**Acceptance criteria:** File exists in data/ root, contains valid JSON, date field points to 2026-05-13, file is parseable by JavaScript fetch.

**Done:** data/active.json created as approval pointer.

### Task 3: Create sample news data in data/2026-05-13/ folder

**Files:** data/2026-05-13/news-01.json through news-07.json

**Action:** Create 7 sample Norwegian tech news JSON files with schema: id, title (2 lines max when rendered), summary (200-300 chars, 5-6 lines in 32-36px font), image_url (HTTPS, publicly accessible), source_url (for QR encoding), source_name, published_at, priority. Content: real Norwegian tech news stories from kode24.no or tldr.tech era 2026-05-13. Each file name matches id field (news-01.json has id "news-01"). Priority 1-7 determines display order. URLs must be valid, images must return 200 OK.

**Acceptance criteria:** 7 JSON files exist in data/2026-05-13/, each contains id, title, summary, image_url, source_url, source_name, published_at, priority fields, all JSON valid, priority 1-7 distributed across files, headlines and summaries in Norwegian, image URLs publicly accessible, no duplicates.

**Done:** Sample news data created for testing carousel.

### Task 4: Create data/2026-05-13/birthday.json optional file

**Files:** data/2026-05-13/birthday.json

**Action:** Create optional birthday card JSON with structure: { "type": "birthday", "names": ["Name1", "Name2"], "message": "Gratulerer med dagen! 🎉" }. This will rotate in carousel alongside news items if present. If file doesn't exist (404), app.js fetch includes try/catch that catches the error and continues without adding birthday card to carousel.

**Acceptance criteria:** File exists with valid JSON, contains type, names array, message field, message in Norwegian, file optional (404 handled gracefully by app.js), carousel rotation includes birthday as additional item if present.

**Done:** Optional birthday.json created for sample data.
