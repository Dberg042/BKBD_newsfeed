# Plan 02-01: Date Handling & Preview Mode Defaults

**Objective:** Add date defaults and ?date URL parameter support to test.html so users can preview news for any date before approval.

**Requirements Addressed:**
- UI-01: test.html preview page defaults to tomorrow''s date
- UI-03: User can override preview date via ?date=YYYY-MM-DD URL param

## Task 1: Implement Date Handling in test.html

**Action:** Add date handling script to test.html with date calculation and URL parameter parsing.

**Acceptance Criteria:**
1. test.html loads without errors
2. Default: page loads tomorrow''s date
3. URL parameter works: test.html?date=2026-05-20 loads news for that date
4. Header displays date; PREVIEW MODE banner shows date
5. Invalid dates fall back to tomorrow without crashing

## Task 2: Verify APP.loadNewsContent Accepts Date Parameter

**Action:** Update APP.loadNewsContent() to accept dateStr parameter and load from data/{dateStr}/ folder.

**Acceptance Criteria:**
1. APP.loadNewsContent() with no args loads from active.json (backward compatible)
2. APP.loadNewsContent(''2026-05-20'') loads from data/2026-05-20/
3. Fetch calls use correct path
4. Malformed JSON items still skip without breaking carousel
5. Missing date folders show fallback message

## Verification Criteria

- test.html loads with no console errors
- Tomorrow''s date displays on first load
- ?date=YYYY-MM-DD URL parameter changes displayed date
- News loads from correct date folder
- Date format conversion is correct (YYYY-MM-DD to DD.MM.YYYY)
