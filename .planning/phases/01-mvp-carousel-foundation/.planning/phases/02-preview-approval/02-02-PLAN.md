# Plan 02-02: Keyboard & Mouse Interactions for Rapid Review

**Objective:** Add arrow key navigation and click-to-pause carousel controls to test.html so users can rapidly review items without waiting for rotation.

**Requirements Addressed:**
- UI-04: Arrow keys skip between items (no 15-second wait)
- UI-05: Click item to pause carousel, click again to resume

## Task 1: Add Keyboard Navigation (Arrow Keys)

**Action:** Add keydown event listener to test.html that responds to arrow left/right keys:
- Arrow Left: skip to previous news item
- Arrow Right: skip to next news item
- Call carousel.prev() and carousel.next() functions
- Reset 15-second timer on each arrow press
- Prevent default key behavior (no page scroll)

**Acceptance Criteria:**
1. Arrow key listeners registered without errors
2. Arrow Left skips to previous item instantly
3. Arrow Right skips to next item instantly
4. Progress bar resets (starts from 0% on new item)
5. Arrow keys work in rapid succession (no debouncing needed)
6. No page scroll or other default behaviors

## Task 2: Add Mouse Click Pause/Resume

**Action:** Add click handler to carousel.item-container element:
- First click: pause carousel (stop 15-second timer)
- Click again: resume carousel (restart 15-second timer)
- Visual feedback: show "PAUSED" indicator or dim progress bar
- Update carousel.js with togglePause() method

**Acceptance Criteria:**
1. Click on any news item pauses carousel
2. Progress bar stops animating
3. Click again resumes carousel
4. Progress bar resumes from current position (not reset)
5. Multiple rapid clicks work correctly (toggle state stable)
6. Pause indicator visible to user (CSS class .paused applied)

## Task 3: Add Pause/Resume Visual Feedback

**Action:** Add CSS styling for paused state:
- .paused class applies to progress bar
- Progress bar appears dimmed or different color when paused
- Optional: show "PAUSED" text overlay or pause icon
- Ensure styling matches dark theme

**Acceptance Criteria:**
1. css/style.css contains .paused class definition
2. Progress bar visually changes when paused
3. Styling consistent with dark theme colors
4. No layout shift when pause state changes
5. Animation restart smooth when resuming

## Verification Criteria

- ✓ Keyboard listeners registered and tested
- ✓ Arrow Left/Right skip items without waiting
- ✓ Click pauses carousel; click resumes
- ✓ Progress bar behavior correct in pause/resume
- ✓ Multiple rapid interactions stable
- ✓ Visual feedback clear to user
- ✓ No regression to Phase 1 carousel behavior
