# Plan 02-03: GitHub Approval Workflow Integration

**Objective:** Implement dynamic Godkjenn button that links to GitHub mobile editor for active.json, allowing users to approve previewed content.

**Requirements Addressed:**
- UI-06: Godkjenn button links to GitHub mobile editor for active.json
- UI-07: User can edit/delete offending news JSON files before approving

## Task 1: Create GitHub URL Generation Function

**Action:** Add getGitHubApprovalUrl(dateStr) function to js/app.js that generates correct GitHub edit link:
- Repository URL: https://github.com/{owner}/{repo}/
- Target file: data/active.json (in main branch)
- Use GitHub''s edit URL format: /edit/main/data/active.json
- Function signature: getGitHubApprovalUrl(dateStr) returns full URL
- Example: https://github.com/kode24-infoskjerm/infoskjerm/edit/main/data/active.json

**Acceptance Criteria:**
1. Function defined in js/app.js
2. Correct GitHub repository URL format
3. Correct file path (data/active.json)
4. Main branch specified (not dynamic)
5. URL opens directly in GitHub editor
6. Mobile view works (GitHub auto-detects mobile)
7. No query parameters or redirects

## Task 2: Wire Godkjenn Button to GitHub URL

**Action:** Update Godkjenn button in test.html:
- Set href to APP.getGitHubApprovalUrl(selectedDate)
- Call on page load and when date changes
- Button text: "Godkjenn" (unchanged)
- Button opens GitHub editor in new tab/window

**Acceptance Criteria:**
1. Godkjenn button has correct GitHub URL
2. URL changes when user changes date (via ?date parameter)
3. Click button opens GitHub editor (target="_blank" or similar)
4. User can edit active.json directly in GitHub
5. User can edit/delete news JSON files in data/{date}/ folder
6. Commit changes directly from GitHub to approve

## Task 3: Document Approval Workflow for Users

**Action:** Add inline documentation and comment in test.html explaining approval workflow:
- How to use preview mode (?date parameter)
- How to navigate (arrow keys, click pause)
- How to approve (click Godkjenn, edit active.json, commit)
- What active.json controls (which date shows on TV)
- Workflow should take under 3 minutes

**Acceptance Criteria:**
1. Comments in test.html explain preview mode
2. Button has title attribute or tooltip describing action
3. README.md updated with approval workflow steps
4. Workflow time estimate documented (under 3 minutes)
5. Example shown: edit active.json date and commit

## Verification Criteria

- ✓ GitHub URL generated correctly for active.json
- ✓ Godkjenn button links to GitHub editor
- ✓ URL updates when date parameter changes
- ✓ User can edit active.json and commit
- ✓ User can delete offending news JSON files
- ✓ Complete approval workflow under 3 minutes
- ✓ Phase 1 production display unchanged
- ✓ test.html still renders identically to index.html

## Threat Model

**Integrity (STRIDE):**
- **I:** User edits active.json on GitHub → GitHub''s version control ensures integrity
- **T:** User approves wrong date → PREVIEW MODE banner clearly warns this is preview
- **D:** User deletes news files accidentally → GitHub shows deletion in commit history
- **R:** User commits without intending to → GitHub requires explicit commit action
