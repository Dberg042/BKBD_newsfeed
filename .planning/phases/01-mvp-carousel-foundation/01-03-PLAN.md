## Objective

Complete error handling, image fallbacks, visual polish, and deployment validation. This plan implements graceful degradation for missing images, network failures, malformed data, and ensures the carousel never displays a blank screen.

## Tasks

### Task 1: Implement image fallback rendering and malformed JSON handling

**Files:** index.html, test.html, css/style.css, js/carousel.js

**Action:** In carousel.js, add image error handler: if image_url returns 404 or fails to load, display source_name as large text (32-36px) in image area instead. In css/style.css, add .image-fallback class styling (centered text, fallback background gradient). In carousel.js, catch JSON parse errors and skip malformed items (log to console, continue to next item). Add fallback message div in index.html for "No news available" state. Update both index.html and test.html to include error message containers: id="no-news-message" (hidden by default, shown if newsItems array is empty). Ensure test.html has identical error containers as index.html for consistent error state testing.

**Acceptance criteria:** Image load failures show source name text fallback, malformed JSON skipped without breaking carousel, fallback message displays if no news items found, carousel continues rotating, no console errors on failure cases, error containers present in both index.html and test.html.

**Done:** Carousel handles missing images and invalid data gracefully.

### Task 2: Implement warning indicator for network failures and retry logic verification

**Files:** js/app.js, css/style.css

**Action:** In app.js, update polling logic to show/hide warning indicator: after 3 consecutive fetch failures to active.json, display #fallback-state div with message "⚠ Tilkoblingsproblem. Viser lagret innhold." In css/style.css, ensure warning indicator styling (yellow-900 bg, orange border, visible text). Add hideErrorIndicator() call when connection restored (fetch succeeds). Verify exponential backoff timing in polling retry logic: 1s delay before retry 1, 3s before retry 2, 9s before retry 3. Test with network disconnect simulation.

**Acceptance criteria:** Warning indicator appears after 3 failed active.json fetches, disappears on successful fetch, error message in Norwegian displays clearly, exponential backoff delays visible in Network tab, last known good content always displayed alongside warning, no blank screen possible.

**Done:** Warning indicator and retry logic working as specified.

### Task 3: Verify deployment readiness and TV compatibility

**Files:** README.md

**Action:** Verify all files are static (no server-side logic). Check that Tailwind CDN and Google Fonts CDN load over HTTPS. Verify no external API calls except GitHub Pages JSON fetch. Create README.md with sections: 1) Project overview, 2) Architecture (data flows from Hermes → git → active.json → TV), 3) Hermes integration contract (daily schedule, input format, output requirements), 4) Deployment: "Push main branch to GitHub, enable GitHub Pages in repo settings, verify index.html loads at domain", 5) Manual approval workflow (user opens test.html, reviews, edits active.json in GitHub mobile UI). Document that all JSON must be valid, all image URLs must return 200 OK, cache-busting (?t=timestamp) used for polling.

**Acceptance criteria:** README.md exists with deployment instructions, no server-side logic in any file, all external resources HTTPS, Hermes contract documented, GitHub Pages deployment steps clear, no build step required, static files only.

**Done:** Project ready for GitHub Pages deployment.
